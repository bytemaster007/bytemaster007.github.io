gulp

gulp

gulp

  • 是一个前端自动化打包构建工具
  • 一个项目在上线之前，要把文件里面的所有空格换行，包括一些不必要的内容去掉
    • es6 语法兼容性不好
    • 上线之前，要把 es6 的代码转换成 es5 的语法
    • …
  • 目前市场上常见的打包工具
    • gulp
      • 用的比较多的 是 3.9.1
      • gulp 4.x.x 19年年初出的
    • webpack
      • webpack 3.x.x
      • webpack 4.x.x
    • 都是对项目进行打包构建
    • 都是基于 node 运行环境

gulp 安装

  • gulp 安装
    • 使用 npm 安装一个全局包就可以了
      • npm i -g gulp
    • 全局 gulp 只是让你电脑有一个环境
      • 可以运行 gulp 指令
  • gulp 安装检测
    • 在命令行使用指令检测
      • 4 以前安装的什么版本就出什么版本号
      • 4 以后出现的是 cli 版本

gulp 使用 前准备

  • 是对你的项目进行打包
  • 你要有一个项目
    • 包含一个开发文件
    • 包含一个生产文件
  1. 建立项目文件夹
    • 项目名 gulp_test
  2. npm 初始化
    • 因为要用 package.json 记录一下我的项目依赖
  3. 建立我的源文件文件夹
    • src 目录是源码
    • gulpfile.js 文件
      • 这个文件里面就来书写一些配置项
        • 怎么打包
      • 这个文件名不能变

gulp 使用前掌握

  • 每一个项目都是独立打包的项目

    • 一个项目一个 gulpfile.js 文件

  • 我们使用gulp的一些功能

    • 需要一个第三方模块
    • 这个第三方模块叫做 gulp
    • 下载 gulp 项目依赖
      • npm i gulp

  • 下载依赖

    • 分两种
      • 一种是项目依赖/生产依赖
        • 平常开发需要用到的包
        • jquery…
      • 一种是开发依赖
        • devDependencies
        • 只用于开发环境，不用于生产环境，项目上线的时候就不需要开发依赖了
        • 打包工具、单元测试之类的都是开发依赖
    • 下载的时候分两种形式下载
      • 开发依赖直接下载
      • 项目依赖 加 –save-dev
        • 简写 加 -D
      • 在 package.json 里面在不同的位置记录
    • 两种下载方式，只是在 package.json 里面记录的位置不一样
      • 为了更直观的看到什么依赖是干什么用的
      • 别的没区别

  • 直接在 gulpfile.js 文件里面导入

    • 第三方模块在导入的时候
    • 直接 require(第三方包名)

  • gulp的工作流程

    • gulp 是一个基于任务的工作流程
    • 我要创建一个 css 的任务
      • 这个任务就来帮我打包 css 文件
      • 当我执行这个 css 任务的时候，就会把我的 css 文件给打包了
      • 我们主要要做的事就是创建任务，让这些任务分别去干什么

  • gulp 常用 API

    • gulp.src()
      • 用来找到文件
      • src(‘a/b/c.css’) // 准确找到一个文件
      • src(‘a/b/**’) // 找到一个文件夹里面的所有文件
      • src(‘a/b/*.css’) // 找打一个文件夹里面的所有 css 文件
      • src('a/b/**/*.css') // 找到一个目录下所有目录里面的css文件
    • gulp.pipe()
      • 就是用来帮你做事的
      • 也叫做管道函数
      • pipe()的参数就是你要做的事情
      • 一个pipe 函数只能做一个事情
    • gulp.dest()
      • 放置文件的
      • 他也是一个事情
      • 使用的时候要在 pipe 里面使用
        • gulp.pipe(gulp.dest())
      • 参数就是你要放置这个文件的 目录地址
    • gulp.series()
      • 用来逐个执行任务的
      • 它里面的每一个参数就是一个任务名称
      • 它里面的多个任务会在前一个任务完成之后执行后一个任务
    • gulp.parallel()
      • 用来并行执行任务
      • 他里面的每一个参数就是一个任务名称
      • 他里面的多个任务会同时开始执行
    • gulp.tesk()
      • 用来创建任务的方法
      • 第一个参数： 任务名称
      • 第二个参数： 是一个函数，函数里面就是这个任务需要做的事情
    • gulp.watch()
      • 用来监控文件的
      • 第一个参数： 文件路径
      • 第二个参数： 当文件改变的时候要执行的任务

gulp 任务的书写

  1. 使用 gulp.task() 定义一个任务
    • 在函数内部进行打包工作
    1. 找到要打包的文件
    2. 使用管道函数执行任务
      • 2.1 放到哪
  2. 在命令行使用指令来让这个任务执行
    • 假设任务名是：css
    • 指令： gulp css 回车

第三方模块

  • gulp-cssmin

    • 专门用来把 css 文件进行压缩处理的
    • 导入以后，你得到的内容就是一个函数，直接让这个函数执行就能压缩 css 文件

  • gulp-autoprefixer

    • 自动给 css 样式添加前缀，用于处理兼容
    • 导入以后得到的就是一个函数，直接让这个函数执行，就能给css添加前缀
      • 需要一些参数

  • gulp-sass

    • 专门把 sass 文件编译成 css 文件使用的
    • 导入以后，你得到的内容就是一个函数
    • 直接调用就能进行编译了
    • 安装 gulp-sass的时候有可能失败
      • 你先安装一个 node-sass
      • 再去安装 gulp-sass 就能成功了

  • gulp-uglify

    • 专门把 js 文件 进行压缩处理
    • 导入以后，你得到的内容就是一个函数
    • 直接调用就行
    • 这个模块不认识es6/es2015

  • gulp-babel

    • 专门用来把es6 转成 es5 语法使用的
    • 但是他的使用需要依赖一些别的包
      • 需要下面两个模块 babel 8的版本才需要下面两个包
        • @babel/core
        • @babel/preset-env
      • babel 8以前的下这两个
        • babel-core
        • babel-preset-env
      • 下载的时候需要下载三个包，使用的时候只需要导入 gulp-babel 就可以了
        • gulp-babel 这个包会去引用上面的两个包
    • npm i -D @babel/core @babel/preset-env
    • gulp-babel 在使用的时候需要传递参数
      • presets:[‘@babel/env’]

  • gulp-htmlmin

    • 专门用来压缩 html 文件使用的
    • 导入以后得到的是一个函数
    • 直接调用 传递一些参数就可以了

  • gulp-file-include

    • 专门用来导入一个html 片段使用
    • 需要传递一些参数
      • prefix: ‘’, // 你定义的标识符
        • 当读取 html 文件时 读取到了这个标识符，就会交给 fileinclude去解析
        • 这个标识符是自定义的
      • basepath:’’, //你存放html片段的那个路径
    • 在你需要导入html 片段的文件里面进行导入
      • 你定义的标识符include(你要导入的html片段名称)
    • 当页面引入的html片段需要根据实际情况微调时
      • 可以在html代码片段里面定义变量
      • 在引入时传入变量值来让其显示不同的内容
      • 语法：
        • 变量： 标识符变量名
        • 引入： 你的标识符include(html片段文件名, json格式的数据)

  • gulp-imagemin

    • 专门用来压缩图片的
    • 压缩力度不会很大
    • 这个包下载很费劲（很大）
    • 使用的时候需要传递一些参数
  • del
    • 专门用来删除文件夹的
    • 使用方式 del(你要删除的文件夹)
  • gulp-webserver
    • 专门帮我们自动打开一个浏览器
    • 打开的文件是哪个文件
      • 打开压缩的文件
    • 使用的时候需要配置一些参数

注意

  1. gulp 打包的时候 不会把原先已经存在的文件给清除掉，只会把新的文件添加进来

    • 需要定义一个清除的任务，在打包之前把先前的文件夹清除掉，再重新添加
    • gulp.task('default', gulp.series('del', gulp.parallel('css', 'sass', 'js', 'html', 'image', 'audio', 'lib', 'data')))

  2. 自动打开浏览器 和 热刷新
